## 1. はじめに

本ツールは、短縮URLの作成・クリック解析・成約トラッキング、および紹介（リファラル）リンクの管理を行うWebアプリケーションです。

### 主な機能

- **短縮URL作成**: 長いURLを短くし、クリック数を計測
- **解析**: 時間帯別・デバイス別・参照元別などの詳細分析
- **A/Bテスト**: 複数の転送先を比率設定で振り分け、自動最適化
- **成約トラッキング**: ピクセルタグでコンバージョンを計測
- **紹介リンク管理**: 紹介者ごとの成約追跡とメール通知
- **QRコード**: 各URLのQRコードを即時生成・保存
- **CSVエクスポート**: クリックデータや紹介者データのダウンロード

### 動作環境

- PHP 8.4 / MariaDB（MySQL）
- Xserver 共有ホスティング対応
- モダンブラウザ（Chrome, Safari, Edge, Firefox）

## 2. インストール・初期設定

### 2.1 ファイル配置

サーバー上の公開ディレクトリにファイル一式を配置します。

```
/public_html/r/     ← アプリケーションルート
├── admin/              ← 管理画面
├── assets/             ← CSS, JS, 画像
├── includes/           ← PHPライブラリ
├── templates/          ← テンプレート
└── install/            ← インストーラー
```

### 2.2 インストーラー実行

ブラウザで `https://あなたのドメイン/r/install/` にアクセスし、以下を入力します。

| 項目 | 説明 |
|------|------|
| DBホスト | データベースサーバー（通常 `localhost`） |
| DB名 | 事前に作成したデータベース名 |
| DBユーザー | データベース接続ユーザー |
| DBパスワード | データベース接続パスワード |
| ベースURL | `https://あなたのドメイン/intro` |
| 管理者ユーザー名 | ログイン用ユーザー名 |
| 管理者パスワード | 8文字以上 |

インストール完了後、`config.php` に設定が書き込まれ、全テーブルが自動作成されます。

### 2.3 ログイン

`https://あなたのドメイン/r/admin/` にアクセスし、設定した管理者アカウントでログインしてください。

- ログイン試行制限: **5回** 失敗でアカウントロック（15分間）
- セッション有効時間: **60分**

## 4. 短縮URL管理

### 4.1 URL作成

**メニュー**: 短縮URL → URL作成

#### 必須項目

| 項目 | 説明 |
|------|------|
| 転送先URL | リダイレクト先のURL（`https://` から始まる完全なURL） |

#### オプション項目

| 項目 | 説明 |
|------|------|
| 名前 | 管理用の名前（未入力の場合スラッグが使用されます） |
| 有効期限 | 期限を過ぎるとエラーページにリダイレクト |
| カテゴリ | 分類用カテゴリ |
| グループ | A/Bテスト用グループ |
| メモ | 管理者向け内部メモ |

#### スラッグタイプ（3種類）

| タイプ | 説明 | 例 |
|--------|------|-----|
| アルファベット | 自動連番（a, b, c, ... z, aa, ab ...） | `/a`, `/ab` |
| ランダム単語 | ランダム文字列を自動生成 | `/xk7m2` |
| カスタム入力 | 任意の文字列を指定 | `/campaign2024` |

> カスタムスラッグに使用できる文字: **英数字、ハイフン（-）、アンダースコア（_）**

#### 転送方式（2種類）

| 方式 | 説明 |
|------|------|
| ジャンプ | 標準リダイレクト（アドレスバーが転送先に変わる） |
| URL保持 | iframe内で表示（アドレスバーは短縮URLのまま） |

URL保持モードの場合、「タイトルバー文字」でブラウザのタブに表示されるテキストを指定できます。

#### 作成結果

作成成功後、4つの形式で短縮URLが表示されます。

| 形式 | 例 |
|------|------|
| default | `https://example.com/r/abc` |
| .htm | `https://example.com/r/abc.htm` |
| .html | `https://example.com/r/abc.html` |
| dir | `https://example.com/r/abc/` |

各URLの右に**コピー**ボタンと**QRコード**ボタンがあります。

### 4.5 短縮URLの利用方法（エンドユーザー向け）

短縮URLを受け取ったエンドユーザー（クリックする側）の利用方法と動作を説明します。

#### 短縮URLの形式

エンドユーザーに渡す短縮URLは以下の4形式があります。どの形式でも同じ転送先に遷移します。

| 形式 | URL例 | 用途の目安 |
|------|------|----------|
| 標準 | `https://ドメイン/r/abc` | 汎用（最短） |
| .htm | `https://ドメイン/r/abc.htm` | メルマガ・広告向け |
| .html | `https://ドメイン/r/abc.html` | メルマガ・広告向け |
| ディレクトリ | `https://ドメイン/r/abc/` | SNS投稿向け |

> `.htm` / `.html` / ディレクトリ形式は、メール配信システムやSNSでURLが途切れる場合の回避策として利用できます。

#### クリック時の動作

**ジャンプモード（標準）の場合**:

1. ユーザーが短縮URLをクリック
2. システムがクリックを記録（IP・ブラウザ・デバイス等）
3. ブラウザが転送先URLに自動リダイレクト（アドレスバーが転送先に変わる）

ユーザーにとっては、リンクをクリックするだけで転送先ページが表示される通常のWebリンクと同じ体験です。

**URL保持モード（iframe）の場合**:

1. ユーザーが短縮URLをクリック
2. システムがクリックを記録
3. ブラウザのアドレスバーは**短縮URLのまま**、ページ全体にiframeで転送先が表示される

アドレスバーに短縮URLを残したい場合（ブランディング目的など）に使用します。

#### エンドユーザーへの配布方法

| 配布手段 | 推奨URL形式 | 補足 |
|----------|------------|------|
| メール本文 | `.html` または `.htm` | URLが途切れにくい |
| LINE / SNS DM | 標準 or ディレクトリ | 短いほうが見やすい |
| SNS投稿 | ディレクトリ | 末尾 `/` でリンクが安定 |
| 紙媒体・チラシ | QRコード | 管理画面からQR画像を保存して利用 |
| Webサイト・ブログ | 標準 | `<a href="短縮URL">テキスト</a>` |

#### 有効期限切れの場合

短縮URLに有効期限が設定されている場合、期限を過ぎるとエラーページにリダイレクトされます（設定画面で指定したエラーページURL、または標準のエラー表示）。

#### ユニーク判定の仕組み

- 同一IPアドレス・同一ブラウザからのアクセスは、**24時間以内に1回**のみユニークとしてカウントされます
- ユーザーが異なるブラウザやデバイスでアクセスした場合は、それぞれ別のユニークとしてカウントされます
- Cookieは使用しません（IPアドレスとブラウザ情報のみで判定）

## 5. 解析

**メニュー**: 短縮URL → 解析

特定の短縮URLのアクセスデータを詳細に分析します。

### 操作手順

1. ドロップダウンから解析対象のURLを選択
2. 日付を選択（日別・時間帯別の場合）
3. 表示モードを選択

### 表示モード（7種類）

| モード | 説明 |
|--------|------|
| 時間帯別 | 0時〜23時のクリック分布 |
| 日別 | 月内の日別クリック推移 |
| 曜日別 | 全期間の曜日別集計 |
| 参照元別 | direct / search / social / email / other |
| デバイス別 | PC / モバイル / タブレット / その他 |
| ブラウザ別 | Chrome, Safari, Firefox 等のTOP20 |
| OS別 | Windows, iOS, Android 等のTOP20 |

### 表示内容

- **サマリー**: 選択日のトータル/ユニーク数
- **内訳テーブル**: 各項目のトータル・ユニーク・割合（プログレスバー付き）
- **棒グラフ**: 時間帯別/日別/曜日別のビジュアル表示

「CSVエクスポート」ボタンで表示中のデータをダウンロードできます。

## 6. グループ（A/Bテスト）

**メニュー**: 短縮URL → グループ

複数の転送先URLに対してトラフィックを分配し、成約率を比較するA/Bテスト機能です。

### グループの作成

| 項目 | 説明 |
|------|------|
| グループ名 | A/Bテストの名前 |
| 転送方式 | ジャンプ or URL保持 |
| メモ | 管理メモ |

### 転送先の設定

「+」ボタンで転送先を追加します。

| 項目 | 説明 |
|------|------|
| ラベル | A, B, C... の識別ラベル |
| 転送先URL | それぞれの転送先 |
| 比率 | トラフィック振り分け比率（例: A:1, B:1 = 50%:50%） |

### 自動最適化ルール

| 項目 | 説明 |
|------|------|
| 成約率の差（%） | この差以上開いたら勝者を自動固定（0=無効） |
| 最低成約数 | 最低限のCV数を達成するまでは固定しない（0=無効） |
| 固定方法 | 「Best（成約率最高）」または「First（最初の転送先）」 |

### ステータス

| 状態 | 説明 |
|------|------|
| テスト中（青） | A/Bテスト進行中 |
| ロック中（緑） | 勝者が決定し、その転送先に固定済み |

### A/B比較テーブル

各転送先について以下が表示されます:

- ラベル / 転送先URL / 比率
- クリック数 / ユニーク / CV数 / CV率
- 🏆: 現在のベストパフォーマー
- 🔒: ロック中の転送先
- CV率比較バー（視覚的な比較）

「ロック」ボタンで手動固定、「ロック解除」でテスト再開が可能です。

### 使い方

1. グループを作成して転送先を2つ以上設定
2. 短縮URL作成/編集時に、このグループを割り当て
3. 短縮URLがクリックされると、比率に応じて転送先が振り分けられる
4. 成約率が一定条件を満たすと、勝者に自動固定される

## 8. 紹介（リファラル）機能

紹介者（アフィリエイター等）ごとに固有リンクを発行し、成約を個別に追跡する機能です。

### 8.1 キャンペーン管理

**メニュー**: 紹介 → キャンペーン

#### キャンペーン作成

| 項目 | 説明 |
|------|------|
| キャンペーン名 | 管理用名称（例: 「春のキャンペーン2026」） |
| スラッグ | URLの一部になる英数字（例: `spring2026` → `/ref/spring2026`） |
| 転送先URL | 紹介リンクのリダイレクト先 |
| 開始日時 | キャンペーン開始日（任意） |
| 終了日時 | キャンペーン終了日（任意） |
| パラメータ渡し | 紹介リンクのパラメータを転送先に引き継ぐ |
| 成約通知メール | 成約時に紹介者へメール通知する（デフォルトOFF） |
| メモ | 管理メモ |

#### キャンペーンカードの表示内容

- **ステータス**: 有効（緑）/ 無効（灰）/ 終了（黄）
- **紹介URL形式**: `/ref/スラッグ?intro=紹介コード`
- **統計**: 総アクセス / ユニーク / 成約 / CV率
- **CVトラッキングタグ**: コピー可能なHTMLタグ

#### CVタグの設置

キャンペーン詳細に表示されるCVタグを、成約ページ（サンクスページ等）に貼り付けてください:

```html
<img src="https://あなたのドメイン/r/ref_cv.php?campaign=スラッグ"
     width="1" height="1" style="display:none">
```

---

### 8.2 紹介者管理

**メニュー**: 紹介 → 紹介者

#### 紹介者の追加（個別）

| 項目 | 必須 | 説明 |
|------|------|------|
| 名前 | ○ | 紹介者の氏名 |
| 紹介コード | ○ | 英数字・ハイフン・アンダースコアのみ（例: `TANAKA`） |
| メール | | 成約通知メールの送信先 |
| グループ | | 組織的な分類ラベル（例: 「東京支部」） |
| メモ | | 管理メモ |

#### 一括追加（CSV形式）

テキストエリアに以下の形式で入力:

```
田中太郎,TANAKA,tanaka@example.com,東京
鈴木花子,SUZUKI,suzuki@example.com,大阪
```

形式: `名前,紹介コード,メール,グループ`（メール・グループは省略可）

#### 紹介者一覧テーブル

| カラム | 説明 |
|--------|------|
| 名前 | 紹介者名 |
| コード | 紹介コード |
| メール | メールアドレス |
| グループ | 所属グループ |
| アクセス | 紹介リンク経由の総クリック数 |
| ユニーク | ユニーク訪問数 |
| CV | 成約数 |
| CV率 | 成約率 |
| 発行URL | 発行済みリンク数（クリックで詳細モーダル） |
| 状態 | 有効/無効 |
| 操作 | 編集・削除ボタン |

#### 検索・フィルタ

- テキスト検索（名前・コード・メール）
- グループフィルタ
- **CSVダウンロード**: フィルタを適用した状態でエクスポート可能

#### 発行済みリンク詳細モーダル

「発行URL」の件数をクリックすると、その紹介者に発行されたリンク一覧が表示されます:

| 項目 | 説明 |
|------|------|
| キャンペーン | 紐づくキャンペーン名 |
| matchコード | バッチ識別子 |
| リンク | 発行済みURL（コピー可能） |
| 状態 | 有効/期限切れ/CP無効/メンバー無効/未開始 |
| 発行日 | リンク生成日 |
| QR | QRコード生成ボタン |

ページネーション: **30件/ページ**

### 8.3 リンク生成

**メニュー**: 紹介 → リンク生成

紹介者に配布する固有リンクを生成します。

#### 生成フォーム

| 項目 | 必須 | 説明 |
|------|------|------|
| キャンペーン | ○ | 有効なキャンペーンから選択 |
| matchコード | | キャンペーンの回次や期間の識別子（例: `202603`） |
| 対象紹介者 | ○ | 個別選択（Ctrl/Cmd+クリックで複数）or「全員分を生成」 |

#### 生成結果

生成されたリンクが一覧表示されます:

| 項目 | 説明 |
|------|------|
| 紹介者 | 紹介者名 |
| コード | 紹介コード |
| メール | メールアドレス |
| 紹介リンク | 生成されたURL |
| コピー | クリップボードにコピー |
| QR | QRコード生成 |

- 「全リンクをコピー」ボタンで全URLを一括コピー
- CSV形式テキストのコピーも可能（Excelに貼り付け可）

#### 発行済みリンク履歴

ページ下部に直近30件の発行履歴が表示されます。各リンクのステータス（有効/期限切れ等）とQRボタン付き。

> リンクは生成時にデータベースに記録されます。同じ組み合わせ（キャンペーン+紹介者+matchコード）で再生成した場合は、URLと日時が更新されます（重複レコードは作成されません）。

### 8.4 紹介リンクの発行・配布ワークフロー（管理者向け）

管理者が紹介者に対してリンクを発行し、渡すまでの一連の手順を説明します。

#### 事前準備

紹介リンクを発行する前に、以下の準備が必要です。

**Step 1: キャンペーンの作成**

1. **紹介 → キャンペーン** を開く
2. 「新規キャンペーン」からキャンペーンを作成
   - **スラッグ**: URLの一部となる英数字（例: `spring2026`）
   - **転送先URL**: 紹介リンクをクリックした人が最終的に到達するページ
   - **開始/終了日時**: キャンペーン期間を設定（任意）
   - **成約通知メール**: ONにすると成約時に紹介者にメール通知
3. キャンペーンを「有効」にする

**Step 2: CVタグの設置**

成約を計測する場合は、転送先のサンクスページ（申し込み完了ページ等）にCVタグを設置します:

```html
<img src="https://あなたのドメイン/r/ref_cv.php?campaign=スラッグ"
     width="1" height="1" style="display:none" alt="">
```

> CVタグを設置しないと成約の計測ができません。先に設置してからリンクを配布してください。

**Step 3: 紹介者の登録**

1. **紹介 → 紹介者** を開く
2. 紹介者を追加（個別入力 or CSV一括入力）
   - **紹介コード**: 各紹介者のユニークな識別子（例: `TANAKA`）
   - **メール**: 成約通知の送信先（通知機能を使う場合は必須）

#### リンク発行手順

**Step 4: リンク生成**

1. **紹介 → リンク生成** を開く
2. 生成フォームに入力:
   - **キャンペーン**: 対象キャンペーンを選択
   - **matchコード**: 発行回や期間の識別子（例: `202603`、`wave1`）。同じキャンペーンで複数回発行する際の区別に使います。省略可
   - **対象紹介者**: 個別選択（Ctrl/Cmdクリックで複数）、または「全員分を生成」にチェック
3. 「リンク生成」ボタンをクリック

**Step 5: 生成結果の確認と配布**

生成が完了すると、紹介者ごとのリンク一覧が表示されます。

配布方法は3つあります:

| 方法 | 手順 |
|------|------|
| **個別コピー** | 各行の「コピー」ボタンで1件ずつクリップボードにコピーし、メール等で送付 |
| **一括コピー** | 「全リンクをコピー」ボタンで全URLをまとめてコピー |
| **CSV形式コピー** | ページ下部のCSV形式テキストをコピーし、Excelに貼り付けて管理・配布 |

各リンクにQRコードボタンもあるため、紙媒体での配布も可能です。

#### 紹介者に伝えるべきこと

紹介者にリンクを渡す際、以下を伝えてください:

1. **配布されたURLをそのまま使うこと** — URLのパラメータ（`intro=`、`sig=` 等）を変更すると正しく計測されません
2. **どのような場面で使うか** — SNS投稿、メール送信、ブログ記事、LINE等
3. **成約の条件** — 何をもって「成約」とするか（申し込み完了、購入完了等）
4. **計測の仕組み** — リンクをクリックしてから72時間以内に成約ページを訪問した場合に記録されます

#### リンクの再発行

同じ組み合わせ（キャンペーン + 紹介者 + matchコード）でリンクを再生成した場合、URLと発行日時が更新されます。レコードが重複することはないため、安心して再発行できます。

#### 発行済みリンクの確認方法

発行済みリンクは以下の場所で確認できます:

| 確認場所 | 確認内容 |
|----------|---------|
| **紹介 → リンク生成** ページ下部 | 直近30件の発行履歴（キャンペーン・紹介者・ステータス） |
| **紹介 → 紹介者** 一覧の「発行URL」列 | 各紹介者の発行リンク数（クリックで詳細モーダル） |

#### ステータスの意味

発行済みリンクには以下のステータスが表示されます:

| ステータス | バッジ色 | 意味 |
|-----------|---------|------|
| 有効 | 緑 | リンクは正常に機能中 |
| 未開始 | 黄 | キャンペーン開始日前 |
| 期限切れ | 灰 | キャンペーン終了日を過ぎた |
| CP無効 | 赤 | キャンペーンが無効化されている |
| メンバー無効 | 赤 | 紹介者が無効化されている |

> 「有効」以外のステータスのリンクをクリックしても、エンドユーザーはエラーページや終了メッセージが表示されます。配布前に必ずステータスを確認してください。

#### 運用の流れまとめ

```
1. キャンペーン作成 → 2. CVタグ設置 → 3. 紹介者登録
→ 4. リンク生成 → 5. 紹介者にリンク配布
→ 6. 成約を紹介解析で確認 → 7. 必要に応じて再発行・追加配布
```

## 9. 成約（CV）トラッキング

成約（コンバージョン）の計測には、成約ページにトラッキングタグを設置します。

### 9.1 短縮URL用CVタグ

#### 推奨: 共通ピクセルタグ

成約ページ（サンクスページ等）に以下を貼り付けます:

```html
<img src="https://あなたのドメイン/r/cv.php"
     width="1" height="1" style="display:none" alt="">
```

このタグは**自動的に**直近72時間以内の同一IP・ユーザーエージェントのクリックを検出し、成約を記録します。

#### URL指定ピクセルタグ

特定の短縮URLに対してのみ成約を記録したい場合:

```html
<img src="https://あなたのドメイン/r/cv.php?slug=abc"
     width="1" height="1" style="display:none" alt="">
```

#### JavaScript形式

```html
<script src="https://あなたのドメイン/r/cv.php?format=js"></script>
```

> **重複防止**: 同一URL + 同一IP + 同一UA の成約は24時間以内に1回のみ記録されます。

---

### 9.2 紹介キャンペーン用CVタグ

#### 共通タグ（推奨）

```html
<img src="https://あなたのドメイン/r/ref_cv.php"
     width="1" height="1" style="display:none" alt="">
```

直近72時間以内の紹介リンク経由の訪問を自動検出し、全対象キャンペーンの成約を記録します。

#### キャンペーン指定タグ

```html
<img src="https://あなたのドメイン/r/ref_cv.php?campaign=スラッグ"
     width="1" height="1" style="display:none" alt="">
```

#### メール通知

キャンペーン設定で「成約通知メール」をONにしている場合、成約が記録されると紹介者のメールアドレスに通知が自動送信されます。

通知メールの内容:
- キャンペーン名
- 紹介者名
- 成約日時
- 累計成約数
